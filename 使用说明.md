# 使用说明（学生节抽奖系统）

这份文档给**现场操作同学**用：照着做即可完成抽奖与大屏展示。

---

## 你会用到的文件

- `index.html`：**控制台**（抽奖按钮都在这里）
- `screen.html`：**大屏展示**（投影/LED/第二屏全屏展示）
- `app.js`：抽奖逻辑（一般不需要改）
- `styles.css`：控制台样式（一般不需要改）
- `背景/`：大屏用的背景图（`1.png` / `2.png` / `3.png`）
- `抽奖素材(1)/`：大屏用的透明叠加素材（按轮次分文件夹）

---

## 快速开始（推荐流程）

### 1) 打开控制台

用 Chrome / Edge / Safari 打开 `index.html`。

### 2) 打开大屏

在控制台点击 **「打开大屏展示」**，会弹出一个新窗口（就是 `screen.html`）。

- 如果没弹出：浏览器可能拦截了弹窗，把本页面设置为“允许弹窗”后重试。
- 大屏窗口建议拖到投影/第二块屏幕，然后按浏览器全屏（常见是 `F11` 或菜单里的“进入全屏”）。

### 3) 按轮次抽奖

控制台里有 4 轮，按按钮抽即可：

- **第一轮（三等奖）**：点按钮抽 3 次字母（会提示第几次）
- **第二轮（二等奖）**：点一次，自动动画后落定 12 个号码
- **第三轮（一等奖）**：点一次，自动生成 32 个结果；控制台可切换“第 1 屏 / 第 2 屏”
- **第四轮（特等奖）**：点一次，落定 1 个结果

> 提醒：系统会自动记忆当前抽奖状态（浏览器缓存）。现场误关页面后，重新打开还能恢复结果。

### 4) 一等奖点名/无人响应处理

- 在控制台第三轮结果里，**点击某个号码**：会在大屏全屏放大显示（用于点名）。
- 再次点击大屏：关闭放大。
- 控制台里 **点中某个号码** 可标记“缺席/无人响应”（会变暗并划线），再次点击可取消。
- **补抽**：第三轮抽完后会出现“补抽 1 个”，用于替换未响应号码。
- 特等奖也有对应的“补抽”。

### 5) 重新开始（清空）

点击 **「重新开始」**：清空所有轮次结果，并同步重置大屏。

## 运行方式（两种都可以）

### 方式 A：直接双击打开（最省事）

直接用浏览器打开 `index.html` 即可。

适合：同一台电脑做控制 + 同一台电脑弹出大屏。

### 方式 B：本地起一个小服务器（更稳）

在项目目录执行（任选一种）：

```bash
python3 -m http.server 8080
```

然后浏览器打开 `http://localhost:8080` 作为控制台。

适合：有些浏览器对“直接打开文件”的权限更严格；起服务会更稳定。

> 注意：目前大屏与控制台的同步方式是 `window.postMessage` + `localStorage`，**最稳的用法仍然是：用控制台按钮弹出大屏窗口**。

## 素材放哪？怎么命名？

大屏 `screen.html` 里写死了素材路径（不需要你改代码，只要目录对上即可）：

- 背景：`背景/1.png`、`背景/2.png`、`背景/3.png`
- 叠加素材根目录：`抽奖素材(1)/`
  - 第一轮：`抽奖素材(1)/第一轮/P.png` … `R.png`
  - 第二轮：`抽奖素材(1)/第二轮/01.png` … `48.png`（两位数补零）
  - 第三轮：
    - 位置素材：`抽奖素材(1)/第三轮/位置1/01.png`、`位置1/P.png` …（位置 1~16）
    - 预案素材（用于放大/特等奖）：`抽奖素材(1)/第三轮/预案/01.png`、`P.png` …
    - 视频：`抽奖素材(1)/第三轮/十连出金.mp4`

如果你现在素材目录名不是 `抽奖素材(1)` 或 `背景`，请把文件夹改名为上述名字（最简单），或让技术同学去改 `screen.html` 里这两行路径常量：

- `const S  = './抽奖素材(1)';`
- `const BG = './背景';`

## “透明素材对不对齐”到底看什么？

大屏里所有叠加 PNG 都是这样显示的：

- 每一张图都会被**铺满整个 16:9 舞台**（`object-fit: fill; width/height: 100%`）
- 系统只是在“铺满”基础上切换透明度（亮/暗），不做自动居中/裁剪

所以对齐的关键是：

- **每一张透明 PNG 的画布尺寸（像素宽高）必须与对应背景一致**  
  比如背景是 1920×1080，那么所有叠加层也建议都是 1920×1080。
- **主体元素（数字/字母）应该放在同一个中心/同一参考点**  
  否则铺满后会出现“看起来偏左/偏右/偏上”的错位。

### 如果现在不对齐，能不能救？

可以，常见两种做法（优先推荐第 1 种）：

- **做法 1：修素材（最稳，现场零风险）**
  - 用 PS/Photopea：新建与背景同尺寸画布（例如 1920×1080）
  - 把主体放到指定位置（通常居中）
  - 导出覆盖原 PNG（保持透明通道）

- **做法 2：改代码做偏移（需要技术同学）**
  - 现在代码是“铺满”，如果要对单张图微调，需要对 `screen.html` 的 `addLayer()` 加偏移/缩放逻辑或针对某些层加 CSS。
  - 适合：只差一点点、但来不及重做整套素材时应急。

## 现场常见问题（快速排查）

- **大屏没反应/不同步**
  - 确认大屏是通过控制台的「打开大屏展示」弹出的
  - 不要用另一台设备单独打开 `screen.html`（当前版本跨设备无法稳定同步）

- **点击“打开大屏展示”没弹窗**
  - 浏览器拦截弹窗 → 允许该站点弹窗后重试

- **抽错了想重来**
  - 点「重新开始」

- **大屏比例不对/被拉伸**
  - 大屏舞台固定 16:9；投影如果不是 16:9 会有拉伸或黑边，这是正常现象
