# 学生节抽奖系统

奖券格式：**字母 (P/O/W/E/R) + 数字 (1–48)**，共 5×48 = 240 张。

## 抽奖轮次（当前实现）

| 轮次 | 奖项 | 规则 |
|------|------|------|
| 第一轮 | 三等奖 | POWER 五个字母中抽 **3 个字母**（逐次抽取，叠图亮灯效果） |
| 第二轮 | 二等奖 | 在 **1-48** 中抽 **12 个数字**（动画过程每帧保持 9 个亮灯） |
| 第三轮 | 一等奖 | 先过场动画，再抽 **32 个字母+数字**；每屏 16 个，分两屏展示，且排除二等奖数字 |
| 第四轮 | 特等奖 | 按预案抽 **1 个字母+数字** |

## 关键规则

- 一等奖号码池会排除所有二等奖号码（即二等奖得主不参与一等奖）。
- 一等奖 32 个结果按两屏显示（16 + 16），且两屏不重复。
- 一等奖结果支持点击号码全屏放大，方便“无人响应”点名。

## 运行方式

### 方式一：直接打开（本机使用）

- 双击 `index.html` 用浏览器打开，或拖入浏览器窗口。
- 大屏展示：点击「打开大屏展示」，会弹出新窗口，可拖到投影/第二屏全屏播放。**控制端与大屏需为同一浏览器同一页面打开**（由控制端弹出的窗口），否则无法同步。

### 方式二：本地服务器（推荐，多设备/大屏）

在项目目录下执行任一命令，再在浏览器访问提示的地址（如 `http://localhost:8080`）。

**Python 3：**
```bash
python3 -m http.server 8080
```

**Node.js（需已安装 npx）：**
```bash
npx -y serve -p 8080
```

**使用启动脚本（已安装 Python 3 时）：**
```bash
./start.sh
# 或
bash start.sh
```

然后在电脑或同一局域网内的手机/平板/大屏设备浏览器中打开：

- 控制端：`http://localhost:8080` 或 `http://你的电脑IP:8080`
- 大屏：由控制端点击「打开大屏展示」弹出的页面，或在新标签页打开 `http://.../screen.html` 并全屏。  
  **注意**：大屏若单独在另一设备打开 `screen.html`，无法接收控制端抽奖结果，需用同一台电脑的控制端弹出大屏，或自行改造为通过后端/WebSocket 同步。

## 部署到其他设备

1. 将整个项目文件夹复制到目标电脑（或 U 盘、网盘）。
2. 在该电脑上按上面「方式二」在项目目录启动本地服务器。
3. 本机访问 `http://localhost:8080` 做控制端；大屏可在此电脑上用「打开大屏展示」全屏，或接投影/第二块屏幕。

若希望手机/平板只做**查看结果**（不操作抽奖），目前需与控制端在同一浏览器会话（由控制端打开大屏窗口）。跨设备实时同步需后续增加服务端或 WebSocket 支持。

## 文件说明

- `index.html` — 抽奖控制页
- `screen.html` — 大屏展示页（3×3 二等奖、一等奖名单等）
- `styles.css` — 样式
- `app.js` — 抽奖逻辑与界面
- `start.sh` — 一键启动本地服务器（默认 8080 端口）

## 现场使用（傻瓜式流程）

### 1) 打开控制台

用 Chrome / Edge / Safari 打开 `index.html`（或本地服务的首页）。

### 2) 打开大屏

在控制台点击 **「打开大屏展示」**，弹出的窗口就是大屏 `screen.html`。

- 如果没弹出：浏览器拦截弹窗，把本页面设置为“允许弹窗”后重试。
- 大屏窗口拖到投影/第二屏后全屏显示（常见 `F11` 或浏览器菜单“进入全屏”）。

### 3) 按轮次抽奖

- **第一轮（三等奖）**：点按钮抽 3 次字母（会提示第几次）
- **第二轮（二等奖）**：点一次，动画后落定 12 个号码
- **第三轮（一等奖）**：点一次，生成 32 个结果；控制台可切换“第 1 屏 / 第 2 屏”
- **第四轮（特等奖）**：点一次，落定 1 个结果

> 系统会自动记忆抽奖状态（浏览器缓存）。误关页面后再次打开可恢复结果。

### 4) 一等奖点名 / 无人响应处理

- **点名放大**：在控制台第三轮结果里点击某个号码 → 大屏全屏放大显示；再点大屏关闭。
- **标记缺席**：控制台第三轮结果里点击某个号码可标记“缺席/无人响应”（变暗并划线），再次点击可取消。
- **补抽**：第三轮抽完后会出现「补抽 1 个」，用于替换未响应号码；特等奖也有补抽。

### 5) 重新开始（清空）

点击 **「重新开始」**：清空所有轮次结果，并同步重置大屏。

## 素材目录与命名（按当前代码实际读取）

大屏 `screen.html` 使用固定相对路径读取素材（默认不需要改代码，只要目录对上即可）：

### 背景（必须）

放在项目根目录的 `背景/` 下：

- `背景/1.png`：第一轮背景（POWER 暗灯底图）
- `背景/2.png`：第二轮背景（号码暗灯底图）
- `背景/3.png`：第三轮/特等奖背景（星空底图）

### 叠加素材（必须）

放在项目根目录的 `抽奖素材(1)/` 下（目录名请保持一致）：

- **第一轮**：`抽奖素材(1)/第一轮/P.png` … `R.png`
- **第二轮**：`抽奖素材(1)/第二轮/01.png` … `48.png`（两位数补零）
- **第三轮**：
  - 位置素材：`抽奖素材(1)/第三轮/位置1/01.png`、`抽奖素材(1)/第三轮/位置1/P.png` …（位置 1~16）
  - 预案素材（用于放大/特等奖）：`抽奖素材(1)/第三轮/预案/01.png`、`抽奖素材(1)/第三轮/预案/P.png` …
  - 过场视频：`抽奖素材(1)/第三轮/十连出金.mp4`

> 如果你的素材文件夹不叫 `抽奖素材(1)` 或 `背景`，最简单是直接改文件夹名；或者改 `screen.html` 里路径常量：`const S = './抽奖素材(1)';`、`const BG = './背景';`。

## 透明 PNG “对齐”标准（你最关心的点）

大屏叠加层（字母/数字 PNG）会被 **强制铺满整个 16:9 舞台**（`width:100%` + `height:100%` + `object-fit: fill`），系统只是切换透明度（亮/暗），不会自动居中或裁剪。

因此对齐的关键是：

- **同一套叠加 PNG 的画布尺寸（像素宽高）要一致**，并且建议与背景图一致（例如都为 1920×1080）。
- **主体内容（数字/字母）要放在同一个参考位置**（通常居中），否则铺满后会出现“看起来偏左/偏右”的错位。

### 不对齐能不能操作？

可以，有两条路（优先推荐第 1 条）：

- **修素材（最稳）**：在 PS/Photopea 新建与背景同尺寸画布，把主体摆正后重新导出 PNG（保留透明通道）。
- **代码偏移（应急）**：需要技术同学改 `screen.html` 的叠加层渲染逻辑，为某些图加偏移/缩放；适合现场只差一点但来不及重做素材时。

## 浏览器建议

推荐使用 Chrome、Edge、Safari 等现代浏览器，以获得最佳显示和弹窗同步效果。
